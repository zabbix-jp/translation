ZABBIX SIA is proud to announce the availability of ZABBIX 1.1.2

ZABBIX is an all-in-one 24x7 monitoring solution without high cost. ZABBIX is released under the GPL, thus it is free of charge for both commercial and non-commercial use. The full license is available at http://www.gnu.org/copyleft/gpl.txt.

This document contains the release notes for ZABBIX 1.1.2. The release does not introduce new functionality, it fixes all known issues of ZABBIX 1.1.1.

The following sections describe the release in detail and provide late-breaking or other information that supplements the main documentation.

What's New in 1.1.2

Support of PostgreSQL

Currently ZABBIX supports three database engines: MySQL, Oracle, PostgreSQL.

Better distribution of refresh moments

Better algorithm of distribution of item next checks is introduced to significantly improve overall performance.

Native ZABBIX agent for 64bit Windows

All Windows 64 bit Intel/AMD platforms are supported.

Misc functionality

  • fixed housekeeper: removal of historical and trend data
  • added Brazilian Portuguese translation
  • added alternative startup scripts for Fedora Linux
  • added Zabbix agent for Win64 platforms
  • added spec file for ZABBIX agent
  • added action email action variable '{TRIGGER.SEVERITY}'
  • improved definition of integer and float point types
  • improved UserParameter substitution
  • disabled exit code handling of UserParameter scripts
  • added color selection for trigger severity 'warning' and 'information'
  • improved macro variables substitution in message body/subject
  • added new value for 'net.tcp.service[*]', 2 - timeout
  • added support of 'pfree' and 'pused' for 'vfs.fs.size' of windows
  • added support of '<=' for trigger severity of action conditions

Fixes

  • fixed storage and calculation of i64 values
  • fixed graph min value calculation
  • fixed caption of IT Services Availability Report
  • fixed graphs drawing
  • fixed incorrect disabling of actions while deleting a trigger
  • fixed incorrect number of problems displaying by submaps
  • fixed warning in trigger selection popup
  • fixed warning in classes.inc.php while displaying graphs
  • fixed errors while adding a host to IT service
  • fixed windows agent crashing with processing of 'system.cpu.util' key
  • fixed Unknown when selecting trigger severity Disaster in action form
  • fixed problem with ASN_IPADDRESS SNMP values
  • fixed 'system.cpu.util[*]' for Tru64
  • fixed problem escaping characters in triggers
  • fixed limitation of return value for 'net.if.*'
  • fixed limitation of return value for 'vfs.dev.*'
  • fixed limitation of return value for 'system.cpu.intr'
  • fixed autoregistration in oracle system
  • fixed order of displaying log files

Installation and Upgrade Notes

Installation

See the Installation Guide http://www.zabbix.com/manual/v1.1/install.php for full details.

Upgrade

In order to upgrade from previous releases, no execution of database patches required. See Upgrade Guide http://www.zabbix.com/manual/v1.1/config_upgrading.php for full details on upgrade procedure.
