ZABBIX SIA is proud to announce the availability of ZABBIX 1.1.1

ZABBIX is an all-in-one 24x7 monitoring solution without high cost. ZABBIX is released under the GPL, thus it is free of charge for both commercial and non-commercial use. The full license is available at http://www.gnu.org/copyleft/gpl.txt.

This document contains the release notes for ZABBIX 1.1.1. The release does not introduce new functionality, it fixes issues some problems of ZABBIX 1.1. The following sections describe the release in detail and provide late-breaking or other information that supplements the main documentation.

The release does not fix support of PostgreSQL. Please expect it in ZABBIX 1.1.2

What's New in 1.1.1

ZABBIX Server

  • added new parameter StatFile for ZABBIX server
  • added startup script for Fedora Core 5

ZABBIX Agent

  • added support of "system.swap.out" for Linux 2.4
  • added support of "system.swap.in" for Linux 2.4

Fixes

  • fixed selection popup if return string contains single quote
  • fixed incorrect characters in item key selector
  • fixed processing of SNMP having non-standard port
  • fixed database data to get rid of zero multiplier for disk spaces
  • misc fixes for WEB frontend
  • fixed srv_status.php: Cannot use object of type CLink ...
  • added removal of stat file on agent shutdown
  • fixed processing of web.page.get and long data sent by agent
  • fixed B64 en-decoding of large data
  • fixed problem with undefined PACKETSZ under Mac OS/X
  • disabled excessive debug printing for system.run
  • umask(022) changed to umask(002)
  • added additional symbols for trigger expressions (slash, space)
  • fixed permissions of log files when starting as 'root'
  • fixed agent crash while evaluating proc.mem[]
  • changed type of lastlogsize to long in structure METRIC
  • ZABBIX agent to bind even if TIME_WAIT sockets exist
  • ZABBIX server to bind even if TIME_WAIT sockets exist
  • fixed notification report for Oracle
  • fixed displayin of long history of alarms
  • fixed value mapping for latest data
  • fixed value mapping for screens

Installation and Upgrade Notes

Installation

See the Installation Guide http://www.zabbix.com/manual/v1.1/install.php for full details.

Upgrade

In order to upgrade from previous releases, no execution of database patches required. See Upgrade Guide http://www.zabbix.com/manual/v1.1/config_upgrading.php for full details on upgrade procedure.

Pre-release testing

ZABBIX Server

ZABBIX Server has been compiled and tested on the following platforms:

  • Ubuntu Linux, AMD64, kernel 2.6.11, MySQL 4.x
  • RedHat AS2 Linux, Intel, kernel 2.4.x, Oracle 9.2.0.4

ZABBIX Agents

ZABBIX Agents have been compiled and tested on the following platforms:

  • AIX 5.2
  • FreeBSD 4.11, 5.4
  • HP-UX 11.00
  • Linux 2.6.11 (Ubuntu)
  • NetBSD 2.0
  • OS/X 10.2
  • Solaris 5.9
  • Tru64 5.1B

Note that agents are available pre-compiled from http://www.zabbix.com/download.php.
