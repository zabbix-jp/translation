ZABBIX SIA is proud to announce the availability of ZABBIX 1.1.5

ZABBIX is an all-in-one 24x7 monitoring solution without high cost. ZABBIX is released under the GPL, thus it is free of charge for both commercial and non-commercial use. The full license is available at http://www.gnu.org/copyleft/gpl.txt.

This document contains the release notes for ZABBIX 1.1.5. The release does not introduce new functionality, it fixes all known issues of ZABBIX 1.1.4.

The following sections describe the release in detail and provide late-breaking or other information that supplements the main documentation.

What's New in 1.1.5

Support of new macros

New macros can be used for messages: {ITEM.NAME}, {PROFILE.DEVICETYPE}, {PROFILE.NAME}, {PROFILE.OS}, {PROFILE.SERIALNO}, {PROFILE.TAG}, {PROFILE.MACADDRESS}, {PROFILE.HARDWARE}, {PROFILE.SOFTWARE}, {PROFILE.CONTACT}, {PROFILE.LOCATION}, {PROFILE.NOTES}, {TRIGGER.URL}, {TRIGGER.COMMENT}

ZABBIX agent can run as 'root'

ZABBIX agent can optionally run with super-user priviledges. Use configuration parameter 'AllowRoot'.

Support of DNS hostname by Windows agent

DNS hostname can be used in parameter Server.

Support of ListenIP by Windows agent

ListenIP is supported by Windows agent.

Improved 'service_status[*]'

Windows agent's 'service_status[*]' accepts both service name and display name.

New minor functionality

  • symbol '$' can be used for hosts and items keys
  • unlimited message size for alerts
  • added sorting by host in trigger status screen
  • added login/logout information into auditlog
  • added filter for disabled items and triggers
  • added support of "vfs.file.exists" key for WinXX
  • www.zabbix.com replaced by localhost in agent
  • added support of "system.swap.(in/out)[swap,param]" for Linux 2.x
  • added "Host group" element for maps
  • improved error messages on startup
  • do not show connector as ON if trigger disabled

Fixes

  • fixed error loging for active checks for Windows agent
  • fixed function max(#N), sum(#N)
  • fixed calculation of aggregate items under Oracle
  • fixed 'eventlog[]' for Win, correctly process event log rotation
  • fixed missing last symbol in vfs.file.regexp
  • fixed 'system.cpu.util[*]' for SunOS
  • fixed permission system
  • fixed rare conflict with cookies of other software
  • fixed displaying of values on overview screen
  • fixed sort order of hosts for trigger overview
  • fixed temporary files removal by WinXX agents
  • added 'with oids' to PostgreSQL table definitions
  • fixed rare incorrect processing of sms messages
  • fixed closing of and printing of stdout/stderr
  • fixed buffer overflow while processing SNMP IP addresses
  • fixed agent's calculation of network statistics
  • fixed some compilation warnings
  • fixed "not equal" condition for hosts and host groups in actions
  • fixed dependences when removing item
  • fixed Y asix value displaying for graphs
  • fixed receipt of hostname on startup of ZABBIX agent
  • fixed 'perf_counter[*]' for WinXX

Installation and Upgrade Notes

Installation

See the Installation Guide http://www.zabbix.com/manual/v1.1/install.php for full details.

Upgrade

You may upgrade from ZABBIX 1.1, 1.1.1, 1.1.2, 1.1.3, 1.1.4 directly to 1.1.5. It requires recompilation of ZABBIX Server and installation of new front-end only. Optionally you may upgrade your agents.

In order to upgrade from previous releases, no execution of database patches required. See Upgrade Guide http://www.zabbix.com/manual/v1.1/install_upgrading.php for full details on upgrade procedure.
